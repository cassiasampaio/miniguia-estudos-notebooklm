# 📘 Miniguia de Estudos: Git e GitHub

## Contexto e Objetivos

Este projeto foi desenvolvido com o objetivo de utilizar Inteligência Artificial como ferramenta de apoio ao aprendizado sobre Git e Github, organizando e sintetizando conteúdos do tema com o uso do [NotebookLM](https://notebooklm.google.com/).

Tema escolhido: **Git e GitHub para iniciantes**

Objetivos:
- Entender os conceitos principais do tema
- Aprender os principais comandos
- Compreender o fluxo de trabalho com GitHub
- Utilizar a IA para estudos de conteúdos técnicos

## Testes de Prompts

### 🔹 Prompt 1
**Pergunta:**
> O que é Git e para que ele serve?

**Resumo da resposta:**
Git é um sistema de controle de versão distribuído que permite acompanhar alterações em arquivos e colaborar em projetos.

---

### 🔹 Prompt 2
**Pergunta:**
> Qual a diferença entre Git e GitHub?

**Resumo da resposta:**
Git é a ferramenta de versionamento local, enquanto GitHub é uma plataforma online que hospeda repositórios Git.

---

### 🔹 Prompt 3
**Pergunta:**
> Explique o fluxo básico de trabalho com Git

**Resumo da resposta:**
O fluxo básico envolve:
- modificar arquivos
- adicionar ao staging (`git add`)
- salvar alterações (`git commit`)
- enviar para o repositório remoto (`git push`)

## 📖 Miniguia de Estudo

### 📌 Resumo

Git é uma ferramenta essencial para controle de versões, permitindo que desenvolvedores acompanhem alterações em arquivos ao longo do tempo.

O GitHub complementa o Git, permitindo armazenar repositórios na nuvem e colaborar com outras pessoas.

---

### 📚 Glossário

- **Repositório:** Local onde o projeto é armazenado
- **Commit:** Registro de uma alteração no projeto
- **Branch:** Linha de desenvolvimento paralela
- **Merge:** Junção de branches
- **Clone:** Cópia de um repositório remoto
- **Push:** Envio de alterações para o repositório remoto
- **Pull:** Atualização do repositório local
